/auth

Interface pour l'inscription et l'authentification.

/auth/ post

Interface pour authentification.

post

Obtenir un jeton d'authentification

post /auth/

Obtenir un jeton d'authentification

  • Request
  • Response

Query Parameters

  • application/json: required (object)

    Identifiants de l'utilisateur

    • username: required (string)

      Nom de l'utilisateur

    • passhash: required (string)

      Mot de passe en clair.

HTTP status code 200

Dans le cas ou le couple login/mot de passe est correct.

Body

Media type: application/json

Type: object

Properties
  • user_id: required (integer)

    ID de l'utilisateur en cours.

  • token: required (string)

    Le jeton d'authentification.

HTTP status code 403

Dans le cas ou le couple login/mot de passe est incorrect.

/auth/new/ post

Interface pour l'inscription

post

Inscrire un compte

post /auth/new/

Inscrire un compte

  • Request
  • Response

Query Parameters

  • creds: required (object)

    Paramètres du compte qu'on veut créer

    • username: required (string)

      Nom d'utilisateur

    • passwd: required (string)

      Mot de passe en clair.

    • email: required (string)

      E-Mail de l'utilisateur

HTTP status code 200

Dans le cas où le compte a été créé correctement.

HTTP status code 500

Dans le cas où la création du compte a échoué.

Body

Media type: application/text

Type: object

post

Mettre à jour la position du joueur.

Mettre à jour la position du joueur.

  • Request
  • Response

Query Parameters

  • position: required (object)

    Player position

    • x: required (integer)
    • y: required (integer)

/questions

Interface pour obtenir les questions environnantes et essayer d'y répondre.

/questions get post

get

Obtenir les questions proches du joueur. Attention, Dans des requêtes GET, on parle exclusivement de paramètres GET.

post

Tenter de répondre à une question

get /questions

Obtenir les questions proches du joueur. Attention, Dans des requêtes GET, on parle exclusivement de paramètres GET.

  • Request
  • Response

Query Parameters

  • user_id: required (integer)

    L'id utilisateur que vous avez obtenu.

  • token: required (string)

    Le jeton d'authentification que vous avez obtenu.

HTTP status code 200

Body

Media type: application/json

Type: object

Properties
  • questions: required (array of Question)

    Items: Question

    • id: required (integer)

      Identifiant unique de la question

    • question: required (string)

      Texte de la question

    • rep1: required (string)

      Réponse 1

    • rep2: required (string)

      Réponse 2

    • rep3: required (string)

      Réponse 3

    • rep4: required (string)

      Réponse 4

    • score: required (integer)

      Score attribué si le joueur répond correctement.

    • difficulty: required (integer)

      Difficulté de la question.

    • position: required (array of integer )

      Position de la question

post /questions

Tenter de répondre à une question

  • Request

Query Parameters

  • answer: required (object)

    Réponse à la question

    • qid: required (integer)

      Identifiant de la question à laquelle on veut répondre.

    • answ_number: required (integer)

      Le numéro de la réponse choisie.

/history/

Obtenir l'historique des dernières questions répondues dans le jeu.

/history/ get

get

Obtenir l'historique des dernières questions répondues dans le jeu. (Le nombre de dernières questions est configurable dans le serveur). Attention, Dans des requêtes GET, on parle exclusivement de paramètres GET.

get /history/

Obtenir l'historique des dernières questions répondues dans le jeu. (Le nombre de dernières questions est configurable dans le serveur). Attention, Dans des requêtes GET, on parle exclusivement de paramètres GET.

  • Request
  • Response

Query Parameters

  • user_id: required (integer)

    L'id utilisateur que vous avez obtenu.

  • token: required (string)

    Le jeton d'authentification que vous avez obtenu.

HTTP status code 200

Body

Media type: application/json

Type: object

Properties
  • history: required (array of HistoryEntry)

    Items: HistoryEntry

    • question: required (string)

      Enoncé de la question.

    • score: required (integer)

      Score apporté par cette question.

    • difficulty: required (integer)

      Difficulté de la question.

    • topic: required (string)

      Domaine de la question.
